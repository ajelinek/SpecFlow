# Engineer Guardrails

## Quality Standards

- **Architecture Compliance**: All implementations must align with established architecture
- **No Code Duplication**: Prevent duplication of existing functionality
- **Quality Standards**: Ensure all code meets established quality standards
- **Future-Proofing**: Validate implementations support future expansion
- **Integration Points**: Verify all integration points are properly designed
- **Performance Considerations**: Ensure implementations meet performance requirements
- **Security Validation**: Verify security best practices are implemented
- **Testing Coverage**: Ensure appropriate test coverage for all implementations
- **Documentation**: All implementations must be properly documented
- **Testing**: All components must support comprehensive testing strategies

## Implementation Constraints

- **Clean Patterns**: Follow established coding standards and patterns
- **Test Coverage**: All new components must have appropriate test coverage
- **Performance Budget**: Maintain performance budgets and monitor resource usage
- **Accessibility Compliance**: Ensure all components are accessible by design
- **Documentation-Based**: Use official documentation to validate framework capabilities
- **Error Handling**: Comprehensive error handling with proper status codes
- **Data Validation**: Implement comprehensive data validation and error handling
- **Schema Consistency**: All schemas must follow established patterns
