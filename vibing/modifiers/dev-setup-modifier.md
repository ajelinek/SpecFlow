# Dev Setup Modifier

**Purpose**: Provides comprehensive guidance for setting up package.json commands to manage the complete development environment, including port management, Firebase emulators, Astro dev server, Playwright E2E tests, and Vitest unit tests.

## Overview

This modifier documents how to configure `package.json` scripts to properly manage the development environment, including killing ports, starting services, running tests, and ensuring all components work together with the dynamic port system.

## Package.json Scripts Architecture

### Core Development Commands

#### 1. Port Management

**Kill Ports:**

```json
{
  "scripts": {
    "kill:ports": "node scripts/kill-firebase-ports.js"
  }
}
```

**Purpose:**

- Kills all processes on instance-specific ports
- Reads `ports-local.json` for port configuration
- Falls back to default ports if config missing
- Used before starting services to prevent conflicts
- Kills ports individually using `spawn` to avoid argument parsing issues

**Usage:**

```bash
pnpm run kill:ports
```

#### 2. Port Configuration

**Setup Ports:**

```json
{
  "scripts": {
    "setup:ports": "node scripts/setup-ports.js"
  }
}
```

**Purpose:**

- Prompts for instance number (1-9)
- Generates `ports-local.json` with instance-specific ports
- Generates `firebase.local.json` with emulator configuration
- Generates `.env` with environment variables

**Usage:**

```bash
pnpm run setup:ports
```

#### 3. Firebase Emulators

**Start Firebase Emulators:**

```json
{
  "scripts": {
    "dev:firebase": "firebase emulators:start --config firebase.local.json"
  }
}
```

**Purpose:**

- Starts all Firebase emulator services
- Uses `firebase.local.json` for port configuration
- Provides local Firebase backend for development

**Usage:**

```bash
pnpm run dev:firebase
```

**What it starts:**

- Firestore Emulator
- Authentication Emulator
- Storage Emulator
- Functions Emulator (only if `functions` directory exists)
- Database Emulator (if configured)
- Emulator UI

**Note:** Functions emulator is automatically excluded if no `functions` directory exists to prevent Firebase startup errors.

#### 4. Astro Dev Server

**Start Astro Dev Server:**

```json
{
  "scripts": {
    "dev:astro": "astro dev"
  }
}
```

**Purpose:**

- Starts Astro development server
- Uses `UI_PORT` from `.env` for port configuration
- Provides hot module reloading
- Serves application at configured port

**Usage:**

```bash
pnpm run dev:astro
```

**Configuration:**

- Reads `UI_PORT` from `.env` (generated by `setup:ports`)
- Port configured in `astro.config.mjs`
- Connects to Firebase emulators via environment variables

#### 5. Combined Development Environment

**Start Complete Dev Environment:**

```json
{
  "scripts": {
    "dev": "pnpm kill:ports && pnpm --parallel --include-workspace-root run \"/^dev.*/\""
  }
}
```

**Purpose:**

- Kills existing ports first (prevents conflicts)
- Starts Firebase emulators and Astro dev server in parallel
- Uses workspace root to include all workspace packages
- Matches all scripts starting with `dev`

**Usage:**

```bash
pnpm run dev
```

**What it does:**

1. Runs `pnpm run kill:ports` to clean up ports
2. Runs all scripts matching `/^dev.*/` in parallel:
   - `dev:firebase` - Starts Firebase emulators
   - `dev:astro` - Starts Astro dev server
   - Any other `dev:*` scripts in workspace

**Benefits:**

- Single command for complete setup
- Automatic port cleanup
- Parallel execution for faster startup
- Works with pnpm workspaces

## Testing Commands

### E2E Tests (Playwright)

**Run E2E Tests:**

```json
{
  "scripts": {
    "test:e2e": "NODE_ENV=test npx dotenv-cli -e .env -- playwright test --project=chromium --reporter=list --retries=1"
  }
}
```

**Purpose:**

- Runs Playwright end-to-end tests
- Loads environment variables from `.env`
- Uses instance-specific ports for test server
- Runs on Chromium with retries

**Usage:**

```bash
pnpm test:e2e
```

**How it works:**

1. Sets `NODE_ENV=test` for test environment
2. Uses `dotenv-cli` to load `.env` file
3. Runs Playwright tests with Chromium
4. Uses `UI_PORT` from `.env` for base URL
5. Playwright config reads `UI_PORT` for webServer URL

**Playwright Configuration Integration:**

**File:** `playwright.config.ts`

```typescript
import * as dotenv from 'dotenv'
dotenv.config()

const uiPort = process.env.UI_PORT || '4000'
const baseUrl = `http://localhost:${uiPort}`

export default defineConfig({
  use: {
    baseURL: baseUrl,
  },
  webServer: {
    command: 'pnpm run dev',
    url: baseUrl,
    reuseExistingServer: !process.env.CI,
  },
})
```

**E2E Test UI Mode:**

```json
{
  "scripts": {
    "test:ui": "NODE_ENV=test npx dotenv-cli -e .env -- playwright test --ui"
  }
}
```

**Purpose:**

- Opens Playwright UI for interactive test debugging
- Loads environment variables from `.env`
- Allows step-by-step test execution

**Usage:**

```bash
pnpm test:ui
```

### Unit Tests (Vitest)

**Run Unit Tests:**

```json
{
  "scripts": {
    "test:unit": "NODE_ENV=test npx dotenv-cli -e .env -- vitest run"
  }
}
```

**Purpose:**

- Runs Vitest unit tests
- Loads environment variables from `.env`
- Tests use Firebase emulator hosts from environment
- Runs tests in data and functions modules

**Usage:**

```bash
pnpm test:unit
```

**How it works:**

1. Sets `NODE_ENV=test` for test environment
2. Uses `dotenv-cli` to load `.env` file
3. Runs Vitest with Node environment
4. Tests connect to Firebase emulators using environment variables

**Vitest Configuration Integration:**

**File:** `vitest.config.ts`

```typescript
import { defineConfig } from 'vitest/config'
import * as dotenv from 'dotenv'
dotenv.config()

export default defineConfig({
  test: {
    environment: 'node',
    include: ['data/**/*.test.ts', 'functions/src/__tests__/**/*.test.ts'],
    globals: true,
    testTimeout: 10000,
  },
})
```

**Component Tests:**

```json
{
  "scripts": {
    "test:component": "NODE_ENV=test npx dotenv-cli -e .env -- vitest run --config vitest.component.config.ts"
  }
}
```

**Purpose:**

- Runs component tests with jsdom environment
- Loads environment variables from `.env`
- Tests SolidJS components

**Usage:**

```bash
pnpm test:component
```

**Component Test Configuration:**

**File:** `vitest.component.config.ts`

```typescript
import { defineConfig } from 'vitest/config'
import * as dotenv from 'dotenv'
dotenv.config()

export default defineConfig({
  test: {
    environment: 'jsdom',
    include: ['src/**/*.test.ts', 'src/**/*.test.tsx'],
    globals: true,
    setupFiles: ['./src/test-setup.ts'],
  },
})
```

## Environment Variable Loading

### Using dotenv-cli

**Why dotenv-cli:**

- Loads `.env` file before running commands
- Ensures environment variables are available to scripts
- Works with all test and dev commands

**Pattern:**

```bash
npx dotenv-cli -e .env -- <command>
```

**Examples:**

- `npx dotenv-cli -e .env -- playwright test` - Loads `.env` for Playwright
- `npx dotenv-cli -e .env -- vitest run` - Loads `.env` for Vitest
- `npx dotenv-cli -e .env -- node script.js` - Loads `.env` for Node script

### Environment Variables in Tests

**Playwright Tests:**

- Read `UI_PORT` from `.env` for base URL
- Use Firebase emulator hosts for test data setup
- Connect to instance-specific emulator ports

**Vitest Tests:**

- Read Firebase emulator hosts from `.env`
- Connect to Firestore, Auth, Storage emulators
- Use instance-specific ports for emulator connections

**Component Tests:**

- Can use Firebase SDK with emulator hosts
- Load environment variables in test setup
- Connect to instance-specific emulator ports

## Development Workflow

### Initial Setup

1. **Install dependencies:**

   ```bash
   pnpm install
   ```

2. **Configure ports:**

   ```bash
   pnpm run setup:ports
   ```

   Enter instance number (1-9)

3. **Start development environment:**
   ```bash
   pnpm run dev
   ```

### Daily Development

1. **Start dev environment:**

   ```bash
   pnpm run dev
   ```

   - Kills existing ports
   - Starts Firebase emulators
   - Starts Astro dev server

2. **Run tests:**

   ```bash
   # Unit tests
   pnpm test:unit

   # Component tests
   pnpm test:component

   # E2E tests (requires dev server running)
   pnpm test:e2e

   # E2E tests with UI
   pnpm test:ui
   ```

3. **Stop services:**
   - Press `Ctrl+C` in terminal
   - Or run: `pnpm run kill:ports`

### Changing Instance Number

1. **Kill existing ports:**

   ```bash
   pnpm run kill:ports
   ```

2. **Reconfigure ports:**

   ```bash
   pnpm run setup:ports
   ```

   Enter new instance number

3. **Restart services:**
   ```bash
   pnpm run dev
   ```

## Test Prerequisites

### E2E Tests (Playwright)

**Requirements:**

- Firebase emulators must be running
- Astro dev server must be running (or Playwright will start it)
- `.env` file must exist with correct ports

**Playwright Auto-Start:**

- Playwright config includes `webServer` that starts `pnpm run dev`
- Only starts if server not already running (`reuseExistingServer: !process.env.CI`)
- Uses `UI_PORT` from `.env` for server URL

**Manual Start:**

```bash
# Terminal 1: Start dev environment
pnpm run dev

# Terminal 2: Run tests
pnpm test:e2e
```

### Unit Tests (Vitest)

**Requirements:**

- Firebase emulators must be running
- `.env` file must exist with correct ports
- Tests connect to emulators using environment variables

**Setup:**

```bash
# Terminal 1: Start Firebase emulators
pnpm run dev:firebase

# Terminal 2: Run unit tests
pnpm test:unit
```

**Note:** Unit tests don't require Astro dev server, only Firebase emulators.

### Component Tests (Vitest)

**Requirements:**

- Firebase emulators must be running (if components use Firebase)
- `.env` file must exist with correct ports
- jsdom environment for DOM testing

**Setup:**

```bash
# Terminal 1: Start Firebase emulators (if needed)
pnpm run dev:firebase

# Terminal 2: Run component tests
pnpm test:component
```

## Best Practices

### Script Organization

1. **Group related scripts:**

   - `dev:*` - Development server scripts
   - `test:*` - Testing scripts
   - `kill:*` - Cleanup scripts

2. **Use consistent naming:**

   - Prefix with category (`dev:`, `test:`)
   - Use descriptive suffixes (`:firebase`, `:astro`, `:e2e`, `:unit`)

3. **Leverage pnpm workspace:**
   - Use `--parallel` for concurrent execution
   - Use `--include-workspace-root` for root scripts
   - Use pattern matching for script groups

### Environment Variable Management

1. **Always use dotenv-cli:**

   - Ensures `.env` is loaded before commands
   - Consistent across all scripts
   - Works with all test frameworks

2. **Set NODE_ENV for tests:**

   - `NODE_ENV=test` for test environment
   - Helps tests identify test mode
   - Can be used for test-specific configuration

3. **Verify .env exists:**
   - Scripts should fail gracefully if `.env` missing
   - Provide clear error messages
   - Guide users to run `setup:ports`

### Port Management

1. **Kill ports before starting:**

   - Prevents port conflicts
   - Ensures clean startup
   - Use in `dev` command

2. **Use instance-specific ports:**

   - Generated by `setup:ports`
   - Stored in `ports-local.json`
   - Used by all services

3. **Handle missing config gracefully:**
   - Fallback to default ports if config missing
   - Provide clear error messages
   - Guide users to run `setup:ports`

## Troubleshooting

### Port Conflicts

**Symptom:** "Port already in use" errors

**Solution:**

```bash
pnpm run kill:ports
pnpm run dev
```

### Missing Environment Variables

**Symptom:** Tests fail or connect to wrong ports

**Solution:**

1. Verify `.env` file exists
2. Check environment variables are correct
3. Re-run `pnpm run setup:ports` if needed

### Tests Can't Connect to Emulators

**Symptom:** Tests fail with connection errors

**Solution:**

1. Verify Firebase emulators are running: `pnpm run dev:firebase`
2. Check `.env` has correct emulator hosts
3. Verify ports match between `ports-local.json` and `.env`

### Playwright Can't Start Server

**Symptom:** Playwright webServer fails to start

**Solution:**

1. Verify `UI_PORT` is set in `.env`
2. Check port is not in use: `pnpm run kill:ports`
3. Verify `pnpm run dev` works manually

## Integration Checklist

After setting up package.json scripts, verify:

- [ ] `pnpm run setup:ports` generates all config files
- [ ] `pnpm run kill:ports` kills processes on instance ports
- [ ] `pnpm run dev:firebase` starts emulators on correct ports
- [ ] `pnpm run dev:astro` starts server on correct port
- [ ] `pnpm run dev` starts both services in parallel
- [ ] `pnpm test:unit` connects to Firebase emulators
- [ ] `pnpm test:e2e` runs E2E tests with correct base URL
- [ ] `pnpm test:ui` opens Playwright UI
- [ ] All tests use environment variables from `.env`
- [ ] All services use instance-specific ports
