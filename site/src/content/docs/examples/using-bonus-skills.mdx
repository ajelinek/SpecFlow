Bonus skills support the workflow. They are useful because they reduce uncertainty, not because they belong in every feature sequence.

## Typical uses

- `agent-context` when a repo needs fresh root `AGENTS.md` guidance, plus nested overlays only where a subtree genuinely differs
- `deep-research` any time web search or web research is needed, especially when freshness matters

`100-domain-knowledge` usually relies on `deep-research` for non-trivial domain work, so `deep-research` is sometimes a dependency of planning rather than just a side skill.
