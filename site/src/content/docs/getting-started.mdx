## 1. Install the skills

Install the main SpecFlow entrypoint:

```bash
npx skills add ajelinek/SpecFlow --skill specflow
```

Install every bundled skill individually too:

```bash
npx skills add ajelinek/SpecFlow --skill '*'
```

<Callout tone="tip" title="Skills install versus agents install">
  `npx skills add ...` installs the SpecFlow skills. The bundled worker agents are a separate optional install.
</Callout>

## 2. Install the optional worker agents

If you want the supporting `coder`, `execution-agent`, and `designer` workers, use **[Custom Agents](./custom-agents/)** next.

<Callout tone="tip" title="When the agents matter most">
  The bundled workers are the intended support structure for full `301`, `302`, and `401` runs. You can start without them, but stronger orchestration comes from installing them.
</Callout>

## 3. Run the lean default

If the repo already has enough context, the lean path is the fastest clean start:

<PromptBlock
  title="Run this"
  text={`Run 201-high-level-design for invoice approval.
Use 202-spec-design for invoice approval with balanced coverage.
Run 301-spec-implementation for the invoice approval feature.
Run 401-cleanup on the changed source files.`}
/>

Run `401-cleanup` again later for changed test files if you want a separate test cleanup pass too.

## 4. Pick the right path when the lean flow is not enough

- Read **[Core Workflow](../core-workflow/)** for the larger lifecycle, groupings, and why the `100` series helps on both new and existing projects.
- Use the homepage flow visualization when you want to compare the full, conservative, lean, and implementation-first paths side by side.

<Callout tone="tip" title="What 301 does when artifacts are missing">
  `301` can synthesize missing `201`, `202`, and `203` outputs in memory and run `204-feature-validation` before coding. Run `203-implementation-design` yourself first when you want a persisted `.specflow/features/<name>/implementation.md` file.
</Callout>

<Callout tone="tip" title="What the worker agents are for">
  Full `301`, `302`, and `401` runs are designed around separate workers: `coder` for scoped code changes, `execution-agent` for test/lint/build validation, and `designer` for UI-affecting design direction.
</Callout>

## What you will get

| Path | Contents |
|---|---|
| `.specflow/features/<name>/overview.md` | Feature description, scope, user journey |
| `.specflow/features/<name>/specs.feature` | Gherkin scenarios |
| Optional: `.specflow/features/<name>/implementation.md` | Implementation plan written by `203-implementation-design` |
